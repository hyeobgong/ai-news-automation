## 1. 프로젝트 개요
본 프로젝트는 매일 쏟아지는 방대한 AI 관련 뉴스를 자동으로 수집, 정리, 요약하여 사용자에게 제공하는 시스템을 구축하는 것을 목표로 합니다.
Google RSS를 통해 뉴스를 수집하고, TF-IDF 알고리즘으로 중복 기사를 제거한 뒤, Google Gemini API를 활용하여 내용을 분석하고 중요도를 평가합니다.

---

## 2. 개발 진행 상황

### 2.1 프로젝트 기획 및 문서화 (`make_ppt.py`)
- **기능**: 프로젝트의 제안서 및 실행 계획을 담은 프레젠테이션(PPT)을 자동으로 생성합니다.
- **주요 내용**:
    - 다크 모드(Dark Mode) 스타일 적용으로 가독성 및 세련미 향상.
    - 슬라이드 구성: 프로젝트 개요, 필요성, 기술 스택, 실행 계획, 활용 방안.
- **결과물**: `AI_News_Auto_Project_Plan.pptx` 파일 생성 완료.

### 2.2 뉴스 데이터 수집기 (`src/rss_collector.py`)
- **기능**: 지정된 키워드(예: "생성형 AI", "인공지능")를 기반으로 Google News RSS 피드에서 최신 기사를 수집합니다.
- **구현 기술**: `feedparser` 라이브러리 활용.
- **주요 로직**:
    - RSS 피드 파싱 후 제목, 링크, 요약문, 발행일 정보 추출.
    - 데이터 구조화 및 리스트 형태 반환.

### 2.3 뉴스 데이터 처리기 (`src/news_processor.py`)
- **기능**: 수집된 뉴스의 중복을 제거하고 AI 모델을 통해 심층 분석을 수행합니다.
- **핵심 알고리즘**:
    1.  **중복 제거 (Deduplication)**:
        - `scikit-learn`의 `TfidfVectorizer` 활용 (Character n-gram 방식 적용).
        - 코사인 유사도(Cosine Similarity)를 계산하여 유사도가 높은(0.6 이상) 기사들을 그룹화.
        - 가장 긴 텍스트를 가진 기사를 대표 기사로 선정.
    2.  **AI 분석 (Gemini Integration)**:
        - Google `genai` 라이브러리(v1.0+) 사용.
        - 기사 분류(기술/비즈니스), 한글 3줄 요약, 중요도 점수 산출.
        - JSON 포맷으로 구조화된 데이터 응답 유도.

## 3. 현재 이슈 및 대응 현황

### 3.1 Google Gemini API 연동 이슈
- **문제 발생**:
    - `gemini-2.0-flash`: 무료 티어 할당량 초과 (429 Resource Exhausted) 발생.
    - `gemini-pro` / `gemini-1.5-flash`: 일부 모델 경로(404 Not Found) 오류 발생 (API 키 권한 또는 버전 문제).
- **대응 조치 (Hotfix)**:
    - **예외 처리 강화**: API 호출 실패 시 프로그램이 중단되지 않도록 `try-except` 블록 보강.
    - **Fallback 메커니즘**: API 오류 발생 시 "더미 데이터(Dummy Data)"를 생성하여 후속 프로세스가 정상적으로 테스트되도록 조치함.
    - **재시도 로직**: 일시적 오류에 대비해 약간의 대기 시간(`time.sleep`) 추가.

### 3.2 중복 제거 정확도
- **현황**: 테스트 데이터(4건) 기준, 유사한 제목임에도 중복으로 인식되지 않는 케이스 발견.
- **원인**: 제목 텍스트가 짧거나 핵심 키워드 외의 단어 차이가 클 경우 유사도가 낮게 측정됨.
- **향후 계획**: 유사도 임계값(Threshold) 조정(0.6 -> 0.4) 또는 제목과 요약문을 결합하여 비교하는 방식으로 개선 예정.
